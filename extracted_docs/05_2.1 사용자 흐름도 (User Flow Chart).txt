[UX] 2.1 사용자 흐름도 (User Flow Chart)
프로젝트명: 영어 퀴즈 기반 NRC 리워드 플랫폼 (가칭: NRC-Walk & Quiz)
버전: v1.0
작성일: 2026. 04. 30
작성자: 제자님 (개발 총괄)
1. 개요 (Overview)
본 문서는 사용자의 앱 진입부터 보상 획득, 최종 매출 전환까지의 흐름을 정의한다. 크게 [A] 온보딩, [B] 잠금화면 루틴, [C] 인앱 메인 활동, [D] 리워드 전환의 4단계로 구분한다.
2. 세부 흐름도 (Detailed Flows)
[A] 온보딩 흐름 (Onboarding Flow)
처음 앱을 접한 사용자가 서비스에 안착하는 과정이다.
앱 실행: 스플래시 화면 노출.
권한 승인: 만보기(신체활동), 잠금화면(다른 앱 위에 그리기), 알림 권한 요청.
로그인/가입: UUID 기반 자동 가입 또는 NRC 회원 정보(매니저 ID 등) 연동.
튜토리얼: 퀴즈 풀고 포인트 받는 법 짧은 가이드 노출.
메인 진입: 대시보드 화면 노출.
[B] 잠금화면 루틴 (Lockscreen Routine) - 핵심 접점
사용자가 폰을 켤 때마다 반복되는 흐름이다.
잠금화면 활성화: 단어 및 퀴즈 힌트 노출.
사용자 액션:
왼쪽 슬라이드: 그냥 잠금 해제 (홈 화면으로 이동).
오른쪽 슬라이드: 퀴즈 풀기 진입.
퀴즈 풀기: 4지선다형 문제 노출.
정답 여부 판단 (FastAPI 서버):
정답: 즉시 포인트 적립 애니메이션 + "보상 2배 받기" 팝업 노출.
오답: 정답 해설 노출 + 소액 포인트 적립.
보너스 광고 시청 (선택): 보상형 동영상 시청 시 추가 포인트 적립.
잠금 해제 완료: 폰 홈 화면으로 이동.
[C] 인앱 메인 활동 (In-App Activities)
앱을 직접 실행하여 더 큰 보상을 노리는 과정이다.
메인 대시보드: 오늘 걸음 수, 현재 포인트, 전체 광고 수익 현황 확인.
오늘의 퀴즈 세트: 10문제 연속 풀기 도전.
학습 리포트: 내가 자주 틀리는 단어 AI 분석 결과 확인.
추천 광고: 사용자의 관심도에 맞는 NRC 제품(건강/통신) 카드 뉴스 시청.
[D] 리워드 전환 흐름 (Reward Conversion)
모은 포인트를 실제 가치로 바꾸는 과정이다.
포인트 상점 진입: 보유 포인트 확인.
상품 선택: 앤텔레콤 충전권, 앤보임 수강 할인권, NRC 제품 교환권 등 선택.
교환 신청: * 서버에서 포인트 차감 및 쿠폰 번호 생성.
NRC 전산 연동용 코드 발급.
사용 완료: NRC 쇼핑몰 혹은 대리점에서 코드 입력 (매출 발생).
3. 예외 상황 처리 (Exception Handling)
네트워크 단절: 퀴즈 정답 데이터를 로컬 DB에 임시 저장 후, 네트워크 복구 시 서버로 일괄 전송.
어뷰징 감지: 비정상적 속도의 정답 입력 시 "잠시 후 다시 시도하세요" 문구 노출 및 계정 일시 정지.
광고 소진 (No Fill): 구글 광고가 없을 경우, 리워드 포인트 대신 '학습 포인트'만 적립해주고 다음 기회 안내.
4. 시각적 요약 (Visual Summary)
[진입] → 잠금화면 퀴즈 → [보상] → 광고 시청 → [축적] → 포인트 확인 → [전환] → NRC 매출 발생

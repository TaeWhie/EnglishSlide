[Technical] 3.4 보안 및 어뷰징 방지 대책서
프로젝트명: 영어 퀴즈 기반 NRC 리워드 플랫폼 (가칭: NRC-Walk & Quiz)
버전: v1.0
작성일: 2026. 04. 30
작성자: 제자님 (개발 총괄)
1. 개요 (Security Overview)
본 문서는 비정상적인 방법으로 리워드를 획득하려는 시도를 차단하고, 시스템의 무결성을 유지하기 위한 다층 보안 전략을 정의한다. 'Zero-Margin' 정책의 지속 가능성을 위해 부정 적립을 0.1% 미만으로 유지하는 것을 목표로 한다.
2. 클라이언트 보안 (App-Side Security)
2.1 코드 난독화 및 변조 방지
난독화: Flutter의 obfuscate 옵션을 사용하여 리버스 엔지니어링을 어렵게 만듦.
무결성 체크: 앱 실행 시 바이너리 변조 여부를 확인하여 조작된 앱(Mod APK 등)은 서버 접속을 차단함.
2.2 가상 환경 및 루팅 감지
환경 검사: 녹스(Nox), 블루스택 등 에뮬레이터 환경 및 루팅/탈옥 기기에서의 실행을 제한함.
디버깅 차단: USB 디버깅 모드가 활성화된 상태에서는 포인트 적립 기능을 비활성화함.
3. 네트워크 및 통신 보안 (Network Security)
3.1 통신 암호화 (SSL/TLS)
모든 API 통신은 **HTTPS(TLS 1.3)**를 적용하여 패킷 스니핑을 방지함.
SSL Pinning: 앱에 서버의 공개키를 미리 심어두어, 중간자 공격(MITM)을 통한 데이터 가로채기를 차단함.
3.2 요청 무결성 검증 (Request Signing)
모든 포인트 관련 요청은 요청 본문(Body)과 타임스탬프를 조합한 HMAC 서명을 포함해야 함.
서버는 이 서명을 검증하여 패킷이 중간에 수정되지 않았음을 확인함.
4. 어뷰징 방지 전략 (Anti-Abusing Strategy)
4.1 서버 사이드 퀴즈 검증 (핵심)
로직: 앱은 오직 사용자가 선택한 번호만 전달하며, 정답 판단 및 포인트 지급 처리는 오직 로컬 백엔드(FastAPI) 내부에서만 수행함.
중복 방지: 동일한 퀴즈 ID에 대해 짧은 시간(예: 3초) 내에 중복 요청이 올 경우 최초 1회만 인정함.
4.2 반응 속도 및 패턴 분석 (Bot Detection)
시간 검증: 퀴즈 노출 후 0.5초 이내에 정답을 맞히는 경우(매크로 의심) 포인트 지급을 보류하고 캡차(CAPTCHA) 노출.
좌표 검사: 버튼 클릭 위치가 매번 동일한 픽셀인 경우 매크로 봇으로 간주함.
4.3 기기 및 IP 기반 제한
1기기 1계정: 기기 UUID를 기반으로 한 기기에서 여러 계정을 돌리는 행위를 차단함.
IP Rate Limiting: 특정 IP에서 비정상적으로 많은 요청이 발생할 경우 1시간 동안 해당 IP의 접근을 제한함.
5. 포인트 지급 통제 로직 (Payout Control)
이상 징후 알림: 특정 사용자가 시간당 평균 획득 포인트의 300%를 초과 달성할 경우 관리자에게 즉시 알림 발송.
수동 승인 단계: 일정 금액(예: 50,000포인트) 이상의 상품 교환 신청 시, 관리자가 적립 로그를 검토한 후 최종 승인 처리.
6. 대응 프로세스 (Incident Response)
감지: 실시간 모니터링 시스템에서 어뷰징 의심 사용자 추출.
분석: Solved_Logs와 Point_Logs를 대조하여 부정 방법 파악.
조치: * 1차: 경고 및 해당 포인트 회수.
2차: 리워드 지급 영구 정지.
3차: 기기 및 IP 영구 차단.
기술적 특이사항: 본 보안 정책은 정기적으로 업데이트되며, AI 모델을 통해 실시간으로 정상 사용자와 봇의 패턴을 학습하여 탐지 정확도를 지속적으로 높여 나갈 계획임.

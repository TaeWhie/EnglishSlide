작성일: 2026. 04. 30

1. 개요 (Overview)
본 문서는 서비스 운영 중 발생할 수 있는 기술적 장애 및 비상 상황에 대비하여, 신속한 복구와 피해 최소화를 위한 표준 운영 절차(Standard Operating Procedure)를 정의한다.
2. 장애 등급 정의 (Severity Levels)
등급
정의
판단 기준
Level 1 (긴급)
서비스 전체 중단
서버 다운, DB 접속 불가, 전체 사용자 포인트 소실 위험.
Level 2 (주요)
핵심 기능 오작동
퀴즈 로딩 불가, 포인트 적립 안 됨, 광고 송출 전면 중단.
Level 3 (보통)
부분 기능 장애
상점 내 특정 상품 교환 오류, 일부 UI 깨짐, 리포트 생성 지연.
3. 상황별 대응 절차 (Scenario-based SOP)
3.1 [Level 1] 백엔드 서버 및 DB 장애
현상: 앱 접속 시 '서버 연결 오류' 메시지 노출 및 무한 로딩.
대응 단계:
진단: Docker 컨테이너 상태 확인 (docker ps) 및 서버 리소스(CPU/RAM) 점검.
조치: FastAPI 서비스 재시작. 문제 지속 시 로컬 DB 연결 상태 및 포트 점검.
복구: 서비스 정상화 확인 후 에러 로그 분석을 통해 원인(메모리 누수 등) 파악 후 패치.
3.2 [Level 2] 광고 플랫폼(애드몹) 송출 장애
현상: 광고가 노출되지 않아 리워드 재원 확보가 불가능한 상황.
대응 단계:
확인: 구글 애드몹 대시보드에서 '정책 위반'이나 '계정 정지' 알림 여부 확인.
조치: * 단순 No-Fill(물량 부족)일 경우: 미디에이션(Unity Ads 등) 가중치 상향 조정.
계정 정지 시: 즉시 앱 내 공지를 띄우고 포인트 적립 비율 하향 조정(재원 보호).
복구: 구글 이의신청 진행 및 대체 광고 네트워크 활성화.
3.3 [Level 2] 대규모 어뷰징 및 해킹 공격
현상: 특정 사용자가 비정상적으로 짧은 시간에 대량의 포인트를 적립함.
대응 단계:
차단: 즉시 해당 IP 및 유저 기기 ID(UUID)를 서버 방화벽에서 블랙리스트 처리.
검수: Point_Logs를 전수 조사하여 부정 적립된 포인트 일괄 회수.
보안 강화: 공격에 사용된 취약점(API 허점 등) 확인 후 코드 수정 및 재배포.
4. 사용자 커뮤니케이션 가이드
장애 발생 시 사용자의 불안감을 해소하고 신뢰를 유지하기 위한 공지 원칙입니다.
즉시 공지: 장애 인지 후 15분 이내에 앱 팝업 또는 푸시 알림으로 "긴급 점검 중" 안내.
투명한 설명: "광고 서버 점검으로 인해 잠시 포인트 적립이 지연되고 있습니다"와 같이 원인을 간결하게 설명.
보상 예고: "점검 완료 후 모든 사용자께 사과 포인트(50P)를 지급해 드릴 예정입니다"와 같은 보상안 제시.
5. 복구 후 사후 관리 (Post-Mortem)
장애 보고서 작성: 발생 원인, 대응 시간, 피해 규모(포인트 오지급 등) 기록.
재발 방지 대책: 동일한 장애가 재발하지 않도록 아키텍처 보완 또는 자동 모니터링 알림 설정 강화.
데이터 무결성 검사: 백업본과 현재 DB를 대조하여 소실된 데이터가 없는지 최종 확인.
